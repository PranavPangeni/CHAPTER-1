Data:

Data refers to raw, unprocessed facts and figures without context. It can be in various forms such as numbers, text, images, or sounds. On its own, data lacks meaning and requires processing to become useful information.

Information:

Information is data that has been processed or organized in a way that adds meaning or value. For example, a list of dates is data, but a calendar is information. Information is crucial for decision-making and problem-solving in various fields.

Features of Information:

  1. Subjectivity: Refers to how personal opinions or bias may influence the information.
  2. Timeliness: Indicates how current or recent the information is, which helps in relevance.
  3. Accuracy: Measures how precise and correct the information is, essential for effective decision-making.
  4. Adequacy: Ensures that the information is sufficient to meet specific needs or goals.

Database and its Purpose:

A database is an organized collection of data, typically stored and accessed electronically. Its primary purpose is to efficiently store, retrieve, and manage data, ensuring consistency, security, and accessibility. Databases support various applications, from business operations to scientific research.

Terminologies in Database:

  1. Table: A collection of related data entries organized in rows and columns.
  2. Field: A single data element or attribute within a table, represented by a column.
  3. Record: A complete set of fields (a row) in a table, representing a single entity.
  4. Tuple: Another term for a record or row in a table.
  5. Object: An instance of a class in object-oriented databases, encapsulating data and behavior.
  6. Keys: Essential components used to uniquely identify records within a table and establish relationships between different tables.

Types of Database Model:

  1. Hierarchical: Organizes data in a tree-like structure with a single parent per record. Itâ€™s fast for one-to-many relationships.
  2. Network: Allows many-to-many relationships with records connected through links, forming a graph structure.
  3. Relational: Stores data in tables (relations), with powerful querying capabilities using SQL.
  4. Entity-Relational (ER): Uses ER diagrams to model data relationships visually before database creation.

Integrity Constraints and Types:

  1. Domain Constraint: Ensures values in a column fall within a predefined domain or set.
  2. Entity Integrity: Ensures each table has a primary key and no part of it is NULL.
  3. Referential Integrity: Ensures foreign key values match existing values in the referenced table or are NULL.
  4. Key Constraint: Ensures uniqueness and identification through primary and candidate keys.

Centralized and Distributed Database:

Centralized: All data is stored in a single central location.

Distributed: Data is spread across multiple locations connected via a network.

Advantages:

  • Improved reliability and availability (in distributed).
  • Faster local access (in distributed).
  • Central control and simplicity (in centralized).

Disadvantages:

  • Single point of failure (centralized).
  • Complex synchronization (distributed).
  • Higher cost and setup (distributed).

Comparison:

While centralized systems are simpler and cost-effective, distributed systems offer better performance and fault tolerance.

Practical Topics: DDL and DML Languages

DDL (Data Definition Language): Includes commands like CREATE, ALTER, DROP used to define or modify database structure.

DML (Data Manipulation Language): Includes SELECT, INSERT, UPDATE, DELETE for manipulating data within the tables.

Past Year Questions of DBMS (2020-2024 and Model Questions)

  1. What is Database and DBMS? List out the advantages and disadvantages of DBMS.

    A database is an organized collection of data that is stored and accessed electronically. It allows for efficient storage, retrieval, and management of information. A Database Management System (DBMS) is software that helps in creating, managing, and manipulating databases. It serves as an interface between the database and its users or application programs.

    Advantages: Reduced redundancy, data consistency, easy access and management, enhanced security, backup and recovery options.

    Disadvantages: High cost, complexity, system failure risk, requires technical expertise, performance overhead for small datasets.

  3. Explain different models of DBMS with advantages and disadvantages:

    1. Hierarchical Model: This model organizes data in a tree-like structure where each record has a single parent and can have multiple children.
      Advantages: Fast access, simplicity.
      Disadvantages: Complex to manage many-to-many relationships.
    2. Network Model: Data is represented as records connected by links in a graph, allowing more complex relationships.
      Advantages: Supports complex relationships.
      Disadvantages: Difficult to design and maintain.
    3. Relational Model: Data is stored in tables (relations), and relationships are represented through foreign keys.
      Advantages: Flexibility, SQL support.
      Disadvantages: Slower performance for very large data sets.

  4. What is a relational database? How is it different from other database models?

    A relational database is a type of database that stores data in tables consisting of rows and columns. It uses relational algebra and SQL for querying and managing data. The relationships between tables are established through primary and foreign keys.

  5. What is data redundancy? How does DBMS help in reducing it?

    Data redundancy occurs when the same data is stored in multiple places. DBMS reduces redundancy through normalization and relational models, ensuring each piece of information is stored once and referenced elsewhere using keys.

  7. Who is Database Administrator (DBA)? What are the major responsibilities of DBA?

    A Database Administrator (DBA) is responsible for managing, securing, and maintaining a database system. Tasks include database design, implementation, user access control, performance tuning, backup, and recovery management.

  8. Define normalization. Explain 1NF, 2NF, and 3NF with suitable examples.

    Normalization organizes data to reduce redundancy.
    1NF: Eliminate repeating groups.
    2NF: Eliminate partial dependency.
    3NF: Eliminate transitive dependency.

    Example:
    Unnormalized: ID, Name, Courses
    1NF: Split courses into separate rows.
    2NF: Separate tables for courses and students.
    3NF: Ensure non-key fields depend only on the primary key.

  9. Explain the terms: primary key, foreign key, and candidate key with examples.

    Primary Key: A primary key is a column or set of columns in a table that uniquely identifies each record. For example, in a Student table, StudentID is a primary key.
    Foreign Key: A foreign key is a column that creates a relationship between two tables by referring to the primary key of another table. For example, CourseID in the Enrollment table may refer to CourseID in the Course table.
    Candidate Key: Field(s) A candidate key is any column or set of columns that can serve as a primary key. For example, in a table with columns StudentID, Email, and PhoneNumber, both Email and StudentID can be candidate keys if they contain unique values.

  10. What is SQL? Explain its components and common functions.

    Structured Query Language (SQL) is a standardized language used to manage relational databases. It enables users to create, read, update, and delete data.
    Components: DDL (CREATE, ALTER), DML (INSERT, SELECT), DCL (GRANT, REVOKE), TCL (COMMIT, ROLLBACK).
